<objective>
Create the annotated, un-pushed `v2.5` git tag as the LAST step of the milestone close — it stamps
the finished, proven COMMITTED tree (CLOSE-02 tag half).

Purpose: Mark the shipped v2.5 milestone with a tag, exactly as v2.1–v2.4 were tagged. The tag is
created only after plan 22-01 has committed the 2.5.0 bump and the owner-signed RESULTS-v2.5.md — and
the preconditions are checked against the COMMITTED contents (what `git tag` actually stamps), not the
working tree, so a present-but-uncommitted file can never produce a tag on an unproven commit. The tag
is additionally bound to DOGFOOD_HEAD: its runtime source must be byte-identical to the commit the
dogfood validated, so the tag cannot claim efficacy-proven while stamping un-dogfooded code.

Output: an annotated `v2.5` tag on the `feat/v2.5` branch tip, created locally and NOT pushed
(D-02). The branch is not pushed or merged either — that is a separate owner action outside this
phase.
</objective>

| dogfooded source (DOGFOOD_HEAD) vs tagged source | A runtime-source commit landed after the dogfood but before tagging would let the tag claim efficacy-proven while stamping un-dogfooded code. |
| local tag → remote | D-02 forbids pushing; the boundary is intentionally not crossed in this phase. |
