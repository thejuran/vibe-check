<objective>
Build the config-surface I/O boundary: a new `scripts/config.py` helper that reads the repo-root `.vibe-check.toml` via `tomllib`, resolves each Phase-30 knob (`thresholds`, `disabled`, `top_model`) through the precedence chain (flag > toml > default), and degrades **per-key** to defaults with collected warnings — and a new `scripts/test_config.py` that locks every fail-safe branch.

This is the milestone keystone (CONFIG-03 is the load-bearing invariant every later v2.8 phase assumes). The helper mirrors `score.py`'s module layout (docstring → constants → pure helpers → `__main__` shim) but **inverts its fail posture**: where `score.py` fails CLOSED on bad input, `config.py` DEGRADES — it must NEVER raise to the orchestrator, because a missing/malformed config must never break a review.

Purpose: Establish the one module allowed filesystem I/O so `score.py` stays pure (its import set is frozen at `{json,hashlib,re,sys}` by an AST test — `tomllib` is illegal there). The orchestrator (Plan 03) calls this helper and gets back resolved per-knob values + warnings.
Output: `plugins/vibe-check/scripts/config.py` (new) + `plugins/vibe-check/scripts/test_config.py` (new). Both pass `python3 -m unittest`.
</objective>

load_config(path, *, flags=None) -> (values: dict, warnings: list[str])

values dict shape (always all three keys present, defaults when absent/invalid):
  {
    "thresholds": dict | None,   # None => score.py band_for uses its built-in 95/80/70 literals.
                                 #   When present: {"critical": int, "warning": int, "medium": int}
    "disabled":   list[str],     # [] when absent. Agent names to skip at dispatch.
    "top_model":  str | None,    # None => orchestrator uses its built-in default (opus).
                                 #   When present: validated to {"opus","fable"} (bogus => None + warning)
  }

warnings: list of human-readable strings, e.g.
  "config: thresholds invalid (not strictly descending) — using default"
  "config: .vibe-check.toml unparseable — using defaults"
  EMPTY list when the file is absent OR fully valid.

__main__ shim (how Plan 03 invokes it):
  REPO_ROOT env var in -> reads $REPO_ROOT/.vibe-check.toml -> prints {"values":..,"warnings":..} JSON on stdout -> exit 0 ALWAYS.
