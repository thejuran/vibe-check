  None  => band_for uses the built-in _DEFAULT_BANDS literals (byte-stable, zero-config path).
  dict  => {"critical": int, "warning": int, "medium": int}, each a score floor:
             score >= critical  -> "critical"
             score >= warning   -> "warning"
             score >= medium    -> "medium"
             else               -> None
  Constraint (validated UPSTREAM in config.py, NOT re-validated here): strictly descending
  (critical > warning > medium), medium >= 70. score.py treats the dict as already-valid and
  defensively falls back to defaults for any non-dict / missing sub-key (isinstance guard) so a
  malformed value can never crash run().
